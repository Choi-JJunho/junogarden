```markdown
# 페어프로그래밍 체크리스트
## 페어와 만나서 해야할 일
- [ ] 페어와 같이 밥먹기!
- [ ] 페어와 처음 만나고 10분동안은 미션 이야기 하지않기!
- [ ] 기술부채 페이지 만들기!
	- 노션, 메모장, 블로그 등 아무거나 좋음.
	- 페어와 특정 지식에서 막히면 기술 부채를 작성해두고 나중에 함께 해결해보자!

##  페어에 대해 알아가는 시간
- [ ] 퇴근시간?
- [ ] 스프링 경험이 있는지?
- [ ] 왜 개발자를 하게 되었는가?
- [ ] 이번 미션 난이도를 스스로 책정해본다면?
- [ ] 빠른 구현 vs 느린 학습
- [ ] 이전 페어들과의 페어 프로그래밍 경험은 어땠는가?
- [ ] 프로그래밍을 진행하는 데 중요하게 생각하는게 있다면?

## 페어프로그래밍 진행과정
- [ ] [TDD 진행] 네비게이터는 의식의 흐름을 경계한다!
- [ ] 8번 교체마다 (한 사람이 4번 드라이빙하면) 5~10분 휴식하기
- [ ] 5분마다 드라이버-네비게이터 교체하기
	- 네비게이터는 드라이버가 클래스 생성 시 저희 TDD 해야죠! 라는 말 해주기
- [ ] 궁금하거나 의견차이로 인한 토론이 필요한 경우 시간을 멈추고 토론하기
	- 주관적인 생각 및 기준이 있다면 굽히지 않고 서로 어필하며 토론을 진행하기!
	- 설득 하는쪽이 이기는걸로!
	- 서로 설득이 안된다면 일단 드라이버의 의견을 따른 뒤 추후 토론으로 이어가기!
```
