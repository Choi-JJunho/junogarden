# 서론

초록 스터디가 4주차에 왔다.
기말고사 시험 일정으로 인해 많은 학습이 진행되지는 못했다.

```markdown
커리큘럼

[ 초록 스터디 by. BCSDLab ]
1. 스터디 온보딩 - 진행방식 소개
2. MVC - MVC Response, CRUD API, ArgumentResolver, 예외처리 [미션 1,2,3,4단계]
3. JDBC - Inmemory DB, JDBCTemplate, JDBCTemplate, SimpleJdbcInsert [미션 5,6,7단계]
4. Spring Core - API~DB 복습, 리팩터링, InnerJoin, 객체 의존, 외래키 [미션 8,9,10단계]
5. 버퍼 기간 (부족했다고 느낀 부분을 추가로 학습하는 시간)
6. 스터디 회고
```

# 4주차

4주차는 버퍼기간으로 지금까지 수행한 학습 및 미션 중 시간에 쫓겨 미처 학습하지 못했던 내용을 정리하는 시간으로 잡았다.

이번주는 시험기간이였다. 시험기간에는 쉬는 주간으로 가져갈까? 했지만 몇몇 인원은 시험기간이 오히려 시간이 남아서.. 딱히 쉬지 않아도 되겠다는 의견도 있었다.
그래서 일단 주차는 수행하되 부담갖지 말고 할 수 있는 인원만 수행하도록 진행했다.

# 오프라인 회의

이번주는 특별하게 오프라인 회의를 진행하기로했다.

현재 초록 스터디 활동 내용을 원티드 커뮤니티에 게시하며 스터디 내용을 활발하게 공유하고있다.
원티드에 재직중이신 동아리 선배님의 지원을 받아 원티드 사내 회의실에서 빌려 스터디 오프라인 회의를 진행할 수 있었다.

> 원티드 커뮤니티로 스터디 활동을 공유중입니다. 많관부~
https://www.wanted.co.kr/community

![](/images/9ba15aa7-8040-4e48-826b-afb32c3e4ecc-image.png)

시간이 가능한 인원들이 모여 오프라인 코드리뷰를 진행했다.
서로가 제출한 PR을 확인하며 왜 이런 코드를 선호하는지, 각자의 주관은 어떤지 등의 이야기를 나눌 수 있었다.

기술적인 내용은 온라인으로 충분히 이야기를 나눌 수 있었지만 서로의 주관에 대한 이야기는 오프라인이 확실히 효과적인 것 같다.

# 후기
